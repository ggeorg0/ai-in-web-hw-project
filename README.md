# Grocery Helper — Список покупок с голосовым вводом.

Приложение для создания списка покупок с помощью голоса. Пользователь надиктовывает фразу (например, «купи помидоры и огурцы»), система распознаёт речь, извлекает названия продуктов и приводит их к словарной форме (лемматизирует). Результат автоматически добавляется в список покупок и отображается в веб-интерфейсе.

Стек: **FastAPI** (асинхронный бэкенд на Python), **Celery** + **Redis** (фоновая обработка аудио), **React + Ant Design** (фронтенд), **PostgreSQL** (хранение данных), **Nginx** (единая точка входа и rate-limiting). Мониторинг через **Prometheus** + **Grafana**. Высокая доступность: по 2 реплики backend и celery-worker с graceful shutdown и балансировкой нагрузки.

## ML модели

Главная фича — контейнер **asr-init**, который при первом запуске автоматически оптимизирует модель распознавания речи GigaAM v3 под конкретную машину с GPU от NVIDIA:

```
Исходная модель (PyTorch)
        │
        ▼
    ONNX FP16          ◄── python run_convert_onnx.py
        │
        ▼
  TensorRT engine       ◄── trtexec --fp16 --memPoolSize=workspace:3072
  (model.plan)
        │
        ▼
  Triton Inference      ◄── высокопроизводительный инференс
    Server
```

**Что это даёт:**

- Запуск **одной командой** (`docker compose up -d`) на любой машине с GPU от NVIDIA.
- Не нужно вручную конвертировать модель, подбирать параметры TensorRT или возиться с версиями CUDA
- Достаточно **≥4 ГБ видеопамяти** (проверено на видеокарте ноутбука RTX 2050).
- При повторных запусках конвертация пропускается — `asr-init` проверяет наличие готового `model.plan` и завершается мгновенно.

В качестве LLM для извлечения и лемматизации продуктов используется **Qwen3-0.6B** со **structured output** (structured JSON), запущенная на **vLLM** — open-source движке для высокопроизводительного инференса языковых моделей.

**Поток обработки голосовой команды:**

1. Фронтенд записывает аудио через MediaRecorder API и отправляет на `POST /api/tasks/voice`.
2. Backend сохраняет аудиофайл, создаёт задачу в БД и ставит её в очередь Celery через Redis.
3. Celery Worker забирает задачу: отправляет аудио в Triton (gRPC) → получает текст.
4. Текст отправляется в vLLM → модель Qwen3-0.6B извлекает продукты и лемматизирует их.
5. Результат (список лемматизированных продуктов) сохраняется в БД и публикуется в Redis Pub/Sub.
6. Фронтенд опрашивает `GET /api/tasks/{id}/status` и при успехе автоматически добавляет продукты в список покупок.

**Две изолированные сети:**
- **`frontend_net`** — nginx, frontend, backend, grafana. Фронтенд и прокси не имеют доступа к БД и Redis.
- **`backend_net`** — backend, postgres, redis, celery-worker, triton, vllm, asr-init, prometheus, grafana.
- Backend является единственным мостом между сетями — выполняет роль API-шлюза.

## Системные требования

| Компонент          | Требование                                           |
|---------------------|-----------------------------------------------------|
| GPU                 | NVIDIA с ≥4 ГБ VRAM (протестировано на RTX 2050)    |
| Драйверы            | NVIDIA driver ≥525 + NVIDIA Container Toolkit       |
| Docker              | Docker Engine ≥24 + Docker Compose                  |
| Свободное место     | **40–50 ГБ** (образы Docker + веса моделей + кэш)   |
| ОЗУ                 | ≥16 ГБ (vLLM + Triton + Celery + системные нужды)   |

**Что занимает место:**
- Docker-образ Triton Inference Server с Torch и ASR-зависимостями: ~15 ГБ
- Docker-образ vLLM с HuggingFace-кэшем модели Qwen3-0.6B: ~10 ГБ
- TensorRT engine (`model.plan`) + ONNX-артефакты: ~3 ГБ
- Остальное — образы backend, frontend, postgres, redis, nginx, prometheus, grafana

При первом запуске дополнительно потребуется место для скачивания модели Qwen3-0.6B из HuggingFace (~1.5 ГБ) в кэш `~/.cache/huggingface`.

## Быстрый запуск

```bash
# 1. Клонировать репозиторий с подмодулем GigaAM
git clone --recurse-submodules <URL-репозитория>
cd grocery-helper

# 2. Запустить все сервисы
docker compose up -d
```

**Первый запуск** займёт 15–30 минут: контейнер `asr-init` скачает чекпойнт GigaAM (если отсутствует), экспортирует модель в ONNX FP16, скомпилирует TensorRT-движок под ваш GPU и скопирует model repository в общий том. Triton стартует только после успешного завершения asr-init. vLLM также требует времени на прогрев — модель Qwen3-0.6B загружается из HuggingFace.

**Последующие запуски** занимают 1–2 минуты — конвертация модели пропускается, так как `model.plan` уже готов.

После запуска:
- Веб-интерфейс: **http://localhost**
- Grafana: **http://localhost:3001** (логин/пароль: `admin` / `admin`)
